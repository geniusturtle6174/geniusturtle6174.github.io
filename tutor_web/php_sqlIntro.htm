於本篇中，我們將利用 phpmyadmin 來對 MySQL 資料庫進行一些操作。 首先，連線到「http://localhost/phpmyadmin」，並用之前設定好的密碼登入(帳號是"root")。 由於若將 php 程式用 root 帳號來連線，在碰到安全性上的問題時，後果會比較嚴重， 所以我們要新創一個使用者。請點選「權限」→「新增使用者」來進行，應該會出現如下的畫面：

其中，關於登入資訊的部分，我們可以：

  • 取一個自己的使用者名稱(不分大小寫，此處以"mytest"為例)
  • 主機請改成"本地"，代表 php 程式要跟資料庫在同一台電腦才可以連線
  • 進行設定密碼與確認密碼

"Database for user"則代表是否要建立資料庫 (Database)， 三個選項依序的意思如下(本篇以選項二進行示範)：

  • None: 不建立資料庫
  • Create database with same name and grant all privileges: 建立一個與使用者名稱相同的資料庫，並賦予所有權限
  • Grant all privileges on wildcard name (username\_%): 把現有的同名資料庫讓該使用者操作

"整體權限"的部分，代表對於其他資料庫的操作權限。 如果這個帳號只是給網頁程式使用，則建議全部都不要勾選。

接下來，我們要建立資料表 (Table)。 事實上，資料庫裡面會包含一或多張資料表， 每張資料表則由一或多筆記錄所組成。 例如，我們可以建立以下的資料表(節錄自 KKBOX 某天之排行榜)：
資料序號 當日排名 前日排名 歌曲名稱 演 唱 者
1 1 1 修煉愛情 林俊傑
2 11 12 存在的力量 朱俐靜
3 17 17 Marry me 蕭敬騰
4 46 42 好難得 丁噹
5 60 57 我會在你身邊 朱俐靜
6 63 64 飛機 林俊傑
7 98 99 只能想念你 蕭敬騰

這個資料表包含五個欄位，七筆記錄。 每個資料表可以選定一個欄位做為主索引(primary key)，主索引的欄位值不能重複。 例如，若有一張資料表記錄了同學的學號姓名，則可以選定學號做為主索引。

  • 型態的簡略說明如下
    • CHAR: 固定長度的字串，後方會用空格補齊，長度極限是 65534 (Byte)
    • VARCHAR: 可變長度的字串，長度極限是 65534 (Byte)
    • TINYINT/SMALLINT/MEDIUMINT/INT/BIGINT: 分別佔用 1/2/3/4/8 個 Bytes
